Security Domains

1. Security and Risk Management

Focuses on defining security goals, risk mitigation, compliance, and business continuity. Risk mitigation means having rules and procedures to quickly reduce risk impact.

2. Asset Security

Secures digital and physical assets at all times and locations.

3. Security Architecture and Engineering

Ensures data security is part of system and network design.

4. Communication and Network Security

Manages and secures physical and wireless networks.

5. Identity and Access Management (IAM)

Ensures proper user access using:

  • Identification
  • Authentication
  • Authorization
  • Accountability

6. Security Assessment and Testing

Tests systems for vulnerabilities using scanning and pen testing.

7. Security Operations

Investigates incidents and applies preventive measures.

8. Software Development Security

Uses secure coding practices in the software development lifecycle.

Security Frameworks and Controls

Frameworks help plan for risks. Controls reduce specific risks:

  • Encryption
  • Authentication
  • Authorization

CIA Triad

Core model for security:

  • Confidentiality
  • Integrity
  • Availability

NIST CSF Framework

  • Identify
  • Protect
  • Detect
  • Respond
  • Recover

OWASP Security Principles

  • Minimize Attack Surface Area
  • Principle of Least Privilege
  • Defense in Depth
  • Separation of Duties
  • Keep Security Simple
  • Fix Security Issues Correctly

Audits and Compliance

Security audits review controls and procedures. They help improve security posture and ensure legal compliance.

Security Tools and Log Analysis

We use logs to detect and mitigate threats. Common logs include:

  • Firewall Logs – connection attempts
  • Network Logs – device and computer activity
  • Server Logs – service and system events

SIEM tools analyze logs in one place and help with dashboards, alerts, and visibility. Example: Splunk.

Playbooks for Incident Response

A playbook provides step-by-step guidance during security incidents. It may include:

  • Action steps
  • Flowcharts or tables
  • Roles and responsibilities
  • Recovery procedures

Used for consistency, speed, and clarity under pressure.

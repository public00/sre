6 Alerting Strategies Overview

When evaluating an alerting strategy, we need to consider the following attributes:

  • Precision – Precision is 100% if every alert corresponds to a real issue (i.e., no false positives).
  • Recall – Recall is 100% if every significant issue triggers an alert (i.e., no false negatives).
  • Detection Time – How long it takes to send a notification after a significant issue begins.
  • Reset Time – How long it takes for the alert to stop firing after the issue is resolved.

Alert Strategies

Fires on any event that threatens the SLO.
Example: If the SLO is 99.9% over 30 days, alert if the error rate over the previous 10 minutes is ≥ 0.1%.
Pros: Fast detection, protects SLOs.
Cons: Low precision, noisy alerts.

Increasing the window size means more of the error budget is consumed before the alert fires.
Detection time formula: ((1 - SLO) / error ratio) * window size.
Pros: Higher precision, captures sustained issues.
Cons: Longer detection time, higher computation cost.

Alerts only if the condition remains above the threshold for some time.
If the metric returns to SLO, the timer resets.
Pros: High precision.
Cons: Poor recall, slower detection for fast incidents.

Burn rate describes how fast the error budget is consumed relative to the SLO.
Detection time formula: ((1-SLO) / error ratio) * window size * burn rate.
Budget consumed formula: (burn rate * window size) / period.
Pros: Good precision, fast detection, efficient.
Cons: Low recall at very high burn rates, long reset times.

Set up several burn rate thresholds:
Examples: 2% in 1h, 5% in 6h, 10% in 3d.
Pros: Flexible, good precision and recall.
Cons: Harder to manage, needs alert suppression.

Add a shorter confirmation window to ensure the incident persists.
Rule: Short window = 1/12 of long window.
Example: Page-level alert for >14x burn rate over 1h and 5min.
Pros: Very high precision and recall.
Cons: Complex configuration, harder to manage.

Considerations for Service Traffic Levels

  • High-traffic services: Multi-burn-rate strategies work best for balancing fast, accurate alerts.
  • Low-traffic services:
    • Generate artificial traffic (synthetic monitoring).
    • Group smaller services together for better alerting.

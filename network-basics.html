Network Architecture

Data Packet Structure

A data packet is the basic unit of communication in a network, similar to a physical letter. It consists of:

  • Header: Contains IP address, MAC address, and protocol number
  • Body: The message being sent
  • Footer: Signals the end of the packet

Key Networking Concepts

  • Bandwidth: The amount of data a device receives per second
  • Packet Sniffing: Capturing and inspecting data packets on a network

TCP/IP Model

The TCP/IP model is a standard framework for data transmission. It has four layers:

  • Network Access Layer: Handles physical transmission; includes ARP, cables, hubs, etc.
  • Internet Layer: Connects networks; uses IP for addressing
  • Transport Layer: Manages data flow with TCP (reliable) and UDP (fast, connectionless)
  • Application Layer: Interfaces with user applications; uses HTTP, FTP, DNS, etc.

Networking Components

  • Port: Software-based channel for communication (like an apartment number)
  • IP Address: Unique identifier for a device on the internet
  • MAC Address: Hardware-specific alphanumeric identifier for network devices

Communication Protocols

  • TCP: Reliable, connection-based (transport layer)
  • UDP: Fast, connectionless (transport layer)
  • HTTP: Web communication, insecure (application layer)
  • DNS: Resolves domain names to IPs (application layer, UDP 53)

Management Protocols

  • SNMP: Monitors and configures network devices (application layer)
  • ICMP: Reports errors, used with ping (internet layer)

Security Protocols

  • HTTPS: Secure version of HTTP (port 443, application layer)
  • SFTP: Secure file transfer over SSH (port 22, application layer)

Security Technologies

  • Firewall: Controls access to/from the network
  • VPN: Hides IP and encrypts traffic for privacy
  • Proxy Server: Acts as a middleman to protect internal services

Subnetting & CIDR

Subnetting divides a network into smaller segments to improve performance and security.

CIDR (Classless Inter-Domain Routing) uses notation like /24 to define network sizes and improve routing efficiency.

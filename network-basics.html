Network architecture

Data packet is a basic unit of communication that travels over from one device to another within the network.

Data package is very similar to physical letter. It contains

  • Header - IP address / MAC address / Protocol Number
  • Body - Message that is being sent
  • Footer - Signal the receiving device that packet is finished.

Bandwidth refers to amount of data a device receives every second.

Packet sniffing is a practice of capturing and inspecting data packets across a network.

TCP/IP model

Transmission control protocol / Internet protocol is a standard communication protocol. It establish connection between devices and allows devices to stream the data. TCP/IP model is a framework used to visualise how data is organised and transmitted across the network.

It has 4 Layers:

  • Network Access Layer - deals with creation of network packets and their transmission over the network. This corresponds to the physical hardware involved in network communication. Hubs, modems, cables are all considered part of this layer. The address resolution protocol ARP is part of the network access layer, where it needs to map IP address to MAC address for local communication.
  • Internet Layer - Focuses how networks connect to each other. Internet addresses are attached to data packets to indicate location of sender and receiver. It uses Internet Protocol to send data to correct destination.
  • Transport Layer - Includes protocols to control the flow of traffic across the network. It uses Transmission Control Protocol (TCP) helps with that, and it contains the port number of intended destination service, which resides in the TCP header. User Datagram Protocol (UDP) doesn’t establish connection, and is used for apps that are not concerned with the reliability of the transmission. It is used mostly for Data streaming, like video or sound.
  • Application Layer - how data packets interact with sending and receiving devices. It uses HTTP, SMTP, SSH, FTP, DNS protocol.

Inspecting and troubleshooting issues on the network security professionals can analyse which layers were impacted by an attack based on what processes were involved in an incident.

Port is a software based location that organises the sending and receiving of data between devices on a network. Imagine it as a apartment number in the building, a mailman knows by that number where to deliver the message.

IP address is a unique string of characters that identifies the location of a device on the internet. Internet service provider assign public IP address to our router. Even if there are multiple devices in one network, they are accessing outside network with one public address.

MAC address is a unique alphanumeric identifier that is assigned to each physical device on a network.

Communication protocols

Communication protocols manage how data is transmitted between devices in a network, including timing and data recovery.

  • TCP (Transmission Control Protocol): A reliable, connection-oriented protocol that establishes a connection via a three-way handshake (SYN, SYN/ACK, ACK). Operates at the transport layer.
  • UDP (User Datagram Protocol): A faster, connectionless protocol that does not guarantee delivery, suitable for time-sensitive transmissions like DNS queries. Also at the transport layer.
  • HTTP (Hypertext Transfer Protocol): An application layer protocol used for communication between web clients and servers. Uses port 80 and is being replaced by the more secure HTTPS (which uses SSL/TLS encryption).
  • DNS (Domain Name System): Translates domain names into IP addresses. Uses UDP on port 53, but can switch to TCP for larger replies. Operates at the application layer.

Management protocols

Management protocols are used to monitor, manage, and optimize network performance, as well as report errors.

  • SNMP (Simple Network Management Protocol): Used for monitoring and managing network devices. It can reset passwords, change configurations, and gather bandwidth usage data. Operates at the application layer.
  • ICMP (Internet Control Message Protocol): Used to report errors and diagnose network issues (e.g., with the "ping" command). It sends messages about transmission problems between devices. Operates at the internet layer.

Security protocols

Security protocols ensure that data is securely transmitted over a network using encryption methods to protect against interception.

  • HTTPS (Hypertext Transfer Protocol Secure): A secure version of HTTP that uses SSL/TLS encryption to protect data between clients and servers. Uses port 443 and operates at the application layer.
  • SFTP (Secure File Transfer Protocol): A secure file transfer protocol that operates over SSH (using TCP port 22), providing encrypted file uploads and downloads—commonly used in cloud storage. Also operates at the application layer.

Firewall is network security device that allows or blocks access.

VPNs - is a network security service that changes your public IP address and hides your virtual location so that you can keep your data private.

Subnetting and CIDR Summary

Subnetting is the division of a network into smaller, logical sub-networks (subnets). It enhances efficiency, security, and performance by isolating traffic within segments.

Subnets help enforce security zones (e.g., controlled, restricted, and DMZ zones) by keeping sensitive data separate from public traffic.

When devices within a subnet communicate, switches keep the data within that subnet, reducing network congestion and improving speed.

CIDR (Classless Inter-Domain Routing)

  • CIDR replaced the older classful addressing system to expand IPv4 address availability.
  • CIDR uses a slash notation (e.g., 198.51.100.0/24), where the /24 indicates the network prefix length (i.e., how many bits are used for the network portion).
  • CIDR improves routing efficiency and supports more flexible subnetting.
  • Tools like IPAddressGuide can help convert between CIDR and standard IPv4 formats for better understanding.

Proxy Servers - Can help us secure internal services. It sits between network and our services. It uses different public address than our services.
